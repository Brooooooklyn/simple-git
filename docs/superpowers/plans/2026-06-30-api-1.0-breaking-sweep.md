# Plan: API 1.0 breaking sweep

Branch: `feat/api-1.0-breaking-sweep` (base `4cdc016`, off `main`).

Goal: freeze a coherent, consistent public API before tagging 1.0. Source-verified
findings from a 3-reviewer audit (git2 0.21 / libgit2 1.9.4). Four conventions the
user locked in: **time → JS `Date`**, **bitflags → raw `number` bits**,
**counts/sizes → `number` (no bigint)**, **add async network I/O (additive)**.

This is a BREAKING release on purpose. Backward compatibility is NOT a goal.

## Global Constraints (bind every task)

- napi-rs v3 over `git2` 0.21. Errors go through `IntoNapiError` (`.convert(msg)` /
  `.convert_without_message()`); never `unwrap`/`expect`/panic on a fallible path.
- **Bitflag surfaces use raw `number`**, OR-able by callers. Keep the named
  `const enum`s but their discriminants MUST equal the real libgit2 bit values.
  Rust input conversion uses `git2::X::from_bits_truncate(n)`; returned bitsets
  use `.bits()`. No single-variant `match` with a lossy `_ =>` arm.
- **All time-bearing public APIs return/accept JS `Date`** (via `chrono::DateTime<Utc>`),
  matching `Commit.time()`. No bare seconds/ms numbers for instants.
- **All counts/sizes are `number`**, not `bigint`. Counts cast `as u32`; byte sizes
  use `i64`/`f64` so they map to `number` (lossless to 2^53 = 9 PB).
- Generated `index.js` / `index.d.ts`: after `yarn build:debug`, commit the REAL
  napi output (`git add index.js index.d.ts`). NEVER hand-edit; never `git checkout`
  them. (See memory `napi-build-loader-drift`.)
- Per-task verify loop (all must pass before reporting DONE):
  `cargo fmt -- --check` · `cargo clippy --all-targets` (deny warnings) ·
  `yarn build:debug` · `yarn test`.
- Tests: ava `.mjs` in `__test__/`. Mutating tests use `os.tmpdir()` repos with
  `git config commit.gpgsign false` AND `git config core.autocrlf false`
  (Windows-safe). NEVER use a caret `^` in an `execSync` git command (cmd.exe eats
  it) — use `git rev-list --parents -n 1 <oid>` etc. Read file content with a
  CRLF-tolerant compare when relevant. (See memory `simple-git-panic-and-errorcode-gotchas`.)
- Update `README.md` for any API whose name/shape/type changes, in the same task.
- Commit messages end with: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

## Tasks

### Task 1 — Returned bitsets → `number` (fixes 2 live bugs)
Files: `src/deltas.rs`, `src/remote.rs`. Live bugs today: `flags()` returns `Binary`
for ~every delta (`deltas.rs:41 _ => DiffFlags::Binary`); cred allowed-types mask
collapses to `Default` (`remote.rs:142 _ => CredentialType::Default`).

- `DiffDelta.flags()` → return `number` (the raw `git2::DiffFlags::bits()`).
- Remove the lossy `impl From<git2::DiffFlags> for DiffFlags` (the `_ => Binary` arm).
  Keep the `const enum DiffFlags` (values 1/2/4/8 already correct) as OR-able constants.
- Add free fn `diffFlagsContains(flags: number, flag: DiffFlags): boolean`
  (`(flags & flag) === flag`), mirroring the existing `credTypeContains`.
- `CredInfo.credType` (object field) and `Cred.credtype()` → `number` (raw
  `CredentialType::bits()`). Remove the lossy `impl From<git2::CredentialType>` `_ => Default`.
- `credTypeContains(credType: number, another: CredentialType): boolean` — operate on
  raw bits now (`(credType & another) === another`).
- DiffFile per-flag booleans (`isBinary/isNotBinary/isValidId/exists`) stay (already correct).
- Tests: a real delta's `flags()` has EXISTS bit set (not Binary); `diffFlagsContains`
  works; an SSH cred-info preserves `USERNAME|SSH_KEY` bits and `credTypeContains(info.credType, CredentialType.SshKey)` is true.

### Task 2 — Input flag params → `number` bits
Files: `src/rev_walk.rs`, `src/repo.rs`, `src/remote.rs`.

- `RevWalk.setSorting(sorting: number)` — was `Sort`. Convert via
  `git2::Sort::from_bits_truncate(n as u32)`. Keep `const enum Sort` (0/1/2/4 correct).
- `Repository.openExt(path, flags: number, ceilingDirs)` — was `RepositoryOpenFlags`.
  Convert via `git2::RepositoryOpenFlags::from_bits_truncate(n as u32)`. **Fix the
  enum discriminants** to real libgit2 bits: `NoSearch=1, CrossFS=2, Bare=4,
  NoDotGit=8, FromEnv=16` (currently sequential 0..4 → silent miscoercion).
- `Remote.updateTips(updateFetchhead: number, ...)` — was `RemoteUpdateFlags`. Convert
  via `from_bits_truncate`. Enum values (1/2) already correct.
- Remove the single-variant `From<…>` match impls for these three (replaced by
  `from_bits_truncate`).
- Tests: `setSorting(Sort.Time | Sort.Reverse)` then a 2-commit walk yields
  oldest-first order; `openExt(path, RepositoryOpenFlags.NoSearch | RepositoryOpenFlags.Bare, [])`
  opens; combos no longer throw/miscoerce.

### Task 3 — Counts/sizes → `number` (drop bigint)
Files: `src/blob.rs`, `src/deltas.rs`, `src/commit.rs`, `src/tree.rs`.

- `Blob.size()` → `number` (return `i64`, was `u64`).
- `DiffFile.size()` → `number` (return `i64`, was `u64`).
- `Commit.parentCount()` → `number` (cast `as u32`, was `usize`).
- `Tree.len()` → **rename to `size()`** AND return `number` (cast `as u32`, was
  `u64`). This task OWNS the Tree.len change; no later task touches it.
- Tests: existing tree/commit/blob tests still pass with `number` (no `n` literals);
  add an assertion that `tree.size()` and `commit.parentCount()` are plain numbers
  (`typeof === "number"`).

### Task 4 — Time → JS `Date`
Files: `src/signature.rs`, `src/blame.rs`, `src/file_modification.rs`, `src/repo.rs`.
RISK to verify FIRST: confirm napi maps `chrono::DateTime<Utc>` on a `#[napi(object)]`
FIELD (not just a method return) to JS `Date`. If object fields cannot be `Date`,
STOP and report (do not silently fall back to ms-number).

- `Signature.when()` → `Date` (was seconds-number). `new Signature(name, email, time: Date)`
  → accept a `Date`, convert to epoch seconds for `git2::Time::new`.
- `BlameHunk.finalTime` → `Date` (was ms-number).
- `FileModification`: `authorTime`/`committerTime` → `Date`; DROP the redundant
  `timestamp` field (it equals `committerTime`).
- `Repository.getFileLatestModifiedDate()` / `getFileLatestModifiedDateAsync()` →
  `Date`. `getFileCreatedDate()` / `getFileCreatedDateAsync()` → `Date`. (HIGHEST
  consumer impact — doc-sites use the ms-number form; intended pre-1.0 break.)
- Update all affected tests (compare via `Date`/`.getTime()`).

### Task 5 — Config getters/setters redesign
File: `src/config.rs`.

- `getStringValue` → `getString`; `setStr` → `setString`.
- `getBool`/`setBool` unchanged.
- `getI32`/`setI32` → `getNumber`/`setNumber` (i32 ↔ `number`).
- `getI64`/`setI64` → `getBigInt`/`setBigInt` (i64 ↔ `bigint`, via
  `napi::bindgen_prelude::BigInt`) — fixes the silent >2^53 truncation.
- Tests: round-trip each type; a `getBigInt` value above `Number.MAX_SAFE_INTEGER`
  survives losslessly.

### Task 6 — `find*` consistency + naming/dedup
File: `src/repo.rs` (+ remove `treeEntryToObject`).

- `findTag`/`findTagByPrefix` → return `Tag | null` (map `git2::ErrorCode::NotFound`
  → `None`, propagate other errors via `.convert`), mirroring `find_branch`.
- Remove `Repository.treeEntryToObject` (keep `TreeEntry.toObject(repo)`).
- `message()` → `mergeMessage()`; `removeMessage()` → `removeMergeMessage()`.
- `remoteWithFetch(name, url, refspect)` → param `refspec`.
- `tagAnnotationCreate` → `tagAnnotation` (coherent with `tag` / `tagLightweight`).
- Tests: `findTag` of a missing oid returns null (no throw); `mergeMessage` round-trips.

### Task 7 — Callback shapes → objects + wire DiffOptions
Files: `src/remote.rs`, `src/repo.rs`, `src/diff.rs`.

- `RemoteCallbacks.pushTransferProgress(cb: (progress: PushTransferProgress) => void)`
  — pass the `PushTransferProgress` object (drop the `ts_args_type` positional
  override). Revives the currently-unreachable interface.
- `RemoteCallbacks.pushUpdateReference(cb: (update: { refname: string; status: string | null }) => void)`.
- `Repository.tagForeach(cb: (tag: { id: string; nameBytes: Buffer }) => boolean)`
  — replace the opaque `[string, Buffer]` tuple.
- Wire `DiffOptions` (currently dead, has `showUnmodified`) into
  `diffTreeToWorkdir(oldTree?, options?: DiffOptions)` and
  `diffTreeToWorkdirWithIndex(oldTree?, options?: DiffOptions)`.
- Tests: push callbacks receive an object with the documented fields; tagForeach
  receives `{id, nameBytes}`; `diffTreeToWorkdir(tree, { showUnmodified: true })` works.

### Task 8 — Async network I/O (additive, non-breaking)
Files: `src/repo.rs`, `src/remote.rs` (+ async task structs, mirror `GitModificationTask`).

- Add `Repository.cloneAsync(url, path, signal?)` (static), `commitAsync(...same args as commit..., signal?)`.
- Add `Remote.fetchAsync(refspecs, fetchOptions?, signal?)`, `pushAsync(refspecs, pushOptions?, signal?)`.
- Use `AsyncTask::with_optional_signal` + AbortSignal, `unsafe impl Send` where needed.
  Keep the existing sync variants.
- Tests: `cloneAsync` of a local bare repo resolves; `pushAsync`/`fetchAsync` against a
  local bare remote behave like their sync counterparts.
